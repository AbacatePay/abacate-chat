# Lovable - Plataforma de Desenvolvimento com IA

## O que é Lovable?

Lovable é uma plataforma de desenvolvimento que utiliza inteligência artificial para gerar aplicações web completas. É uma ferramenta que permite criar projetos web modernos usando tecnologias como React, Next.js, TypeScript e outras tecnologias frontend populares.

## Características Principais do Lovable

### 1. **Desenvolvimento Assistido por IA**
- Gera código automaticamente baseado em descrições em linguagem natural
- Cria componentes React funcionais
- Implementa funcionalidades complexas com poucas instruções

### 2. **Stack Tecnológico**
- **Frontend**: React, Next.js, TypeScript
- **Styling**: Tailwind CSS
- **Estado**: Context API, Redux (quando necessário)
- **Deploy**: Integração com plataformas de hospedagem

### 3. **Funcionalidades**
- Criação de componentes reutilizáveis
- Implementação de APIs e integrações
- Geração de formulários e validações
- Criação de interfaces responsivas

## Como Lovable se Relaciona com AbacatePay?

### Integração Possível
Lovable pode ser usado para criar aplicações web que integram com a AbacatePay através de:

1. **Chamadas HTTP diretas** para a API REST da AbacatePay
2. **Componentes React** para checkout e pagamentos
3. **Formulários** para coleta de dados de clientes
4. **Dashboards** para gerenciamento de cobranças

### Exemplo de Uso
```javascript
// Exemplo de como usar AbacatePay em um projeto Lovable
const createPayment = async (paymentData) => {
  const response = await fetch('https://api.abacatepay.com/v1/billing/create', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${process.env.ABACATEPAY_API_KEY}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify(paymentData)
  });
  
  return await response.json();
};
```

## Quando Usar Lovable com AbacatePay?

### Cenários Ideais
- **E-commerce**: Criar lojas online com pagamentos PIX
- **SaaS**: Aplicações que precisam de cobrança recorrente
- **Marketplace**: Plataformas que conectam vendedores e compradores
- **Serviços**: Aplicações que oferecem serviços pagos

### Vantagens da Combinação
- **Desenvolvimento Rápido**: Lovable acelera a criação da interface
- **Integração Simples**: AbacatePay oferece API REST fácil de usar
- **Pagamentos Seguros**: PIX é um método de pagamento confiável
- **Custo Baixo**: AbacatePay cobra apenas R$ 0,80 por transação

## Limitações e Considerações

### Lovable
- Focado em frontend (React/Next.js)
- Requer conhecimento básico de JavaScript/TypeScript
- Pode ter limitações em funcionalidades muito específicas

### AbacatePay
- Apenas pagamentos PIX (não aceita cartão de crédito)
- Necessita de integração manual (não há SDK oficial para Lovable)
- Requer configuração de webhooks para notificações

## Alternativas de Integração

### 1. **API REST Direta**
- Usar fetch() ou axios para chamadas HTTP
- Implementar tratamento de erros manualmente
- Criar componentes customizados

### 2. **Bibliotecas HTTP**
- Axios para requisições mais robustas
- React Query para cache e sincronização
- SWR para data fetching

### 3. **Componentes Customizados**
- Criar componentes React específicos para AbacatePay
- Implementar formulários de checkout
- Desenvolver dashboards de cobrança

## Como Dar Comandos Corretos para o Lovable

### Instruções para o Usuário
Quando você perguntar sobre integração com Lovable, o chatbot deve fornecer comandos específicos que você pode copiar e colar diretamente no Lovable. Use o formato:

**"Para criar isso no Lovable, diga: [comando específico]"**

### Comandos Básicos para Configuração

#### 1. Configurar Variáveis de Ambiente
```
Configure as seguintes variáveis de ambiente no projeto:
- NEXT_PUBLIC_ABACATEPAY_API_KEY: sua_chave_da_api_aqui
- NEXT_PUBLIC_ABACATEPAY_BASE_URL: https://api.abacatepay.com/v1
- NEXT_PUBLIC_ABACATEPAY_DEV_MODE: true
```

#### 2. Criar Serviço Base da AbacatePay
```
Crie um arquivo services/abacatepay.js com uma classe AbacatePayClient que:
- Tenha constructor que recebe apiKey e devMode
- Tenha método makeRequest para fazer chamadas HTTP
- Use fetch() com headers de Authorization Bearer
- Trate erros de resposta da API
- Retorne JSON das respostas
```

### Comandos para Componentes

#### 3. Componente de Checkout
```
Crie um componente React CheckoutForm.jsx que:
- Tenha formulário com campos: nome, email, telefone, CPF/CNPJ
- Use useState para gerenciar estado dos campos
- Tenha validação básica dos campos obrigatórios
- Use Tailwind CSS para styling responsivo
- Tenha botão "Finalizar Compra" com loading state
- Chame API da AbacatePay para criar cobrança
- Redirecione para URL de pagamento retornada
```

#### 4. Componente de Status do PIX
```
Crie um componente PixStatus.jsx que:
- Receba pixId como prop
- Use useEffect para fazer polling a cada 5 segundos
- Verifique status do PIX via API da AbacatePay
- Mostre diferentes estados: PENDING, PAID, EXPIRED
- Use Tailwind CSS para cores diferentes por status
- Tenha loading state durante verificação
```

#### 5. Hook para Gerenciar Pagamento
```
Crie um hook useAbacatePay.js que:
- Gerencie estado de loading, error e success
- Tenha função executeRequest para executar chamadas
- Trate erros automaticamente
- Retorne { loading, error, executeRequest }
```

### Comandos para Páginas

#### 6. Página de Checkout
```
Crie uma página checkout.jsx que:
- Use o componente CheckoutForm
- Tenha layout responsivo com Tailwind CSS
- Mostre resumo do pedido/carrinho
- Tenha breadcrumb de navegação
- Trate redirecionamento após sucesso
```

#### 7. Página de Sucesso
```
Crie uma página success.jsx que:
- Mostre confirmação de pagamento
- Tenha botão "Voltar ao Início"
- Mostre detalhes da transação
- Use design celebrativo com Tailwind CSS
```

### Comandos para Funcionalidades Específicas

#### 8. Criar Cliente
```
Crie função createCustomer que:
- Receba dados do cliente (name, email, cellphone, taxId)
- Valide campos obrigatórios
- Faça POST para /v1/customer/create
- Retorne dados do cliente criado
- Trate erros de validação
```

#### 9. Criar Cobrança
```
Crie função createBilling que:
- Receba dados da cobrança (products, customerId, returnUrl, completionUrl)
- Valide estrutura dos produtos
- Faça POST para /v1/billing/create
- Retorne URL de pagamento
- Trate erros de API
```

#### 10. Criar PIX QR Code
```
Crie função createPixQrCode que:
- Receba amount, description, customer data
- Faça POST para /v1/pixQrCode/create
- Retorne QR code e dados do PIX
- Trate expiração do QR code
```

### Comandos para Melhorias UX

#### 11. Componente de Loading
```
Crie componente LoadingSpinner.jsx que:
- Tenha animação de loading com Tailwind CSS
- Seja reutilizável em diferentes tamanhos
- Tenha texto opcional personalizável
- Use skeleton loading para melhor UX
```

#### 12. Componente de Error
```
Crie componente ErrorMessage.jsx que:
- Mostre mensagens de erro de forma amigável
- Tenha botão para tentar novamente
- Use cores de erro do Tailwind CSS
- Seja dismissible pelo usuário
```

#### 13. Validação de Formulário
```
Crie utilitário de validação que:
- Valide CPF/CNPJ brasileiro
- Valide formato de email
- Valide formato de telefone brasileiro
- Retorne mensagens de erro específicas
- Use regex para validações
```
